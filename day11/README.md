# 📂 HTML/CSS 레이아웃 및 Flexbox 활용 연습

학원 수업 과정에서 웹 페이지의 핵심 레이아웃 구조를 분석하고, 현대 웹 전반에 쓰이는 **Flexbox(`display: flex`)** 정렬 기법을 집중적으로 실습한 저장소입니다. 의미 있는 마크업 구조 정의와 효율적인 스타일 재사용에 초점을 맞추어 연습했습니다.

---

## 🛠️ 주요 학습 및 구현 내용

### 1. 웹 표준에 맞는 시맨틱 마크업 (Semantic Markup)
- `<div>`와 `<section>`의 명확한 역할 분리
  - 단순 디자인 레이아웃 및 CSS 박스 바인딩 목적: `div` 사용
  - 제목(`h1~h6`)과 독립적인 주제를 포함하는 하나의 온전한 문단 구획: `section` 활용
- 스크린 리더 및 SEO를 위한 구조적 마크업 고려 (`.blind` 클래스를 통한 화면 숨김 텍스트 처리)

### 2. Flexbox 핵심 정렬 메커니즘 마스터
- 정렬 기준이 되는 **부모(Flex Container)**와 정렬 대상인 **자식(Flex Item)** 간의 종속 관계 이해
- **주축 정렬(`justify-content`)**: `space-between`과 `center`를 활용하여 로고-내비게이션 양끝 배치 및 중앙 정렬 구현
- **교차축 정렬(`align-items: center`)**: 수직 상하 정렬을 통해 요소들의 높낮이 균형 맞춤
- **역방향 정렬(`flex-direction: row-reverse`)**: HTML 마크업 순서를 변경하지 않고, CSS 제어만으로 글씨와 이미지의 좌우 시각적 배치를 반전시키는 실무 테크닉 습단

## 💻 파일별 실습 요약

### 📄 1. TRAVEL 페이지 레이아웃 (`flex.html`)
- **주요 구현**: 상단 헤더(`header`)와 네비게이션바(`gnb`) 영역의 가로 정렬 및 간격(`gap`) 제어
- **카드 리스트**: 본문 하단의 카드 그리드를 구현할 때, 전체 너비에서 여백을 제외한 비율을 `calc()` 함수로 정산하여 반응형 레이아웃의 기초를 다짐
  - `width: calc((100% - 60px) / 3);`

### 📄 2. 셔클 솔루션 카드 섹션 (`solution.html`)
- **주요 구현**: 서비스 소개 레이아웃 구현
- **문제 해결 경험**: 
  - 기본 `display: flex` 선언 시 요소들이 가로로 정렬되어 레이아웃이 찢어지는 현상을 `flex-direction: column` 설정을 통해 세로 축으로 안정적으로 재배치함.
  - 마크업 구조 상 `이미지 -> 텍스트` 순서로 배치된 요소를 `row-reverse` 속성을 통해 `텍스트 -> 이미지` 형태로 뒤집어 디자인 시안의 좌우 균형을 완벽히 재현.

### 📄 3. 셔클 메인 인트로 페이지 (`shycle.html`)
- **주요 구현**: 실제 기업의 랜딩 페이지 구조 분석 및 클론 코딩 기틀 마련
- **레이아웃 세분화**: 전체를 감싸는 `.wrap` 내부에 고정폭 중앙 정렬을 위한 `.inner` 컴포넌트를 설계하여 구조의 독립성을 확보하고 유지보수성을 향상시킴.

## 💡 느낀 점 및 고찰
- 처음에는 눈으로 코드를 볼 때 다 이해가 가더라도 막상 백지 상태에서 레이아웃을 짤 때 정렬이 깨지는 뇌정지 구간을 경험함.
- 다양한 실습을 통해 플렉스는 **"정렬하고자 하는 자식들이 아니라, 그들을 감싸고 있는 부모 울타리(Container)에 선언해야 비로소 통제된다"**는 본질적인 원리를 명확하게 깨달음.
- 구조적이고 고유한 클래스 이름을 미리 선언해두면 복잡한 부모-자식 자손 선택자 엮기 없이도 단독 선언만으로 단단하고 유연한 CSS 설계가 가능하다는 트렌드를 체득함.
