# 🏛️ DDP(동대문디자인플라자) 웹사이트 클론 코딩 프로젝트

DDP 공식 홈페이지의 주요 레이아웃과 콘텐츠 디자인을 정교하게 구현한 웹 퍼블리싱 개인 토이 프로젝트입니다.

## 📅 프로젝트 기간
* **작업일**: 2026년 5월 28일 (1일간 진행)

---

## 🛠️ 사용 기술 및 스택
* **HTML5** (시맨틱 마크업 설계)
* **CSS3** (Flexbox 레이아웃, 포지셔닝)

---

## 💻 주요 구현 내용 및 학습 포인트

### 1. DDP 프로그램 메인 레이아웃 (`DDP3.html`)
* **카드형 구조 설계**: Flexbox(`display: flex; justify-content: space-between;`)를 적극적으로 활용하여 5개의 프로그램 홍보 카드가 화면 폭에 맞춰 균등하고 일정한 간격(`gap: 20px`)으로 나열되도록 구현했습니다.
* **텍스트 생략 처리법 터득**: 카드의 제목 영역이 길어져 전체 레이아웃이 깨지는 현상을 방지하기 위해 CSS의 `text-overflow: ellipsis;` 속성을 활용, 긴 제목은 말줄임표(...)로 깔끔하게 처리하는 예외 처리를 적용했습니다.

### 2. 아기상어 상세 페이지 정보 레이아웃 (`아기상어.html`)
* **좌우 2단 화면 분할**: 메인 포스터(좌)와 전시 상세 정보(우) 구역을 직관적으로 통제하기 위해 큰 컨테이너 안에 별도의 독립된 방(`info-box`)을 만들어 깔끔한 반반 구조 레이아웃을 잡았습니다.
* **테이블 구조 마스터**: 일정, 장소, 시간 등의 데이터를 반응형 구조로 맞추기 위해 기존의 단순 리스트 정렬에서 표준 `<table>` 태그 세트(`tr`, `th`, `td`) 구조로 업그레이드했습니다. 이를 통해 내용물이 여러 줄로 넘어가도 수직 라인이 칼같이 유지되는 안정적인 정렬을 완성했습니다.
* **절대 좌표 정렬**: 조회수 아이콘과 텍스트 영역을 부모 박스 기준 `position: absolute;`로 띄워 우측 상단 끝에 자연스럽게 안착시켰습니다.

## 🔥 작업하면서 느낀 점 & 해결한 문제
* HTML 태그들을 처음에 크게 묶어두지 않으면 CSS로 가로 분리나 배치를 제어하기가 까다로워진다는 것을 뼈저리게 느꼈습니다. '왼쪽 방'과 '오른쪽 방'으로 감싸서 묶는 구조의 중요성을 확실히 배웠습니다.
* 테이블 마크업 시 `<tr>` 행 단위를 생략하면 브라우저 정렬이 완전히 깨진다는 점을 직접 겪어보며 올바른 웹 표준 태그 사용법의 중요성을 깊이 이해하게 되었습니다.
